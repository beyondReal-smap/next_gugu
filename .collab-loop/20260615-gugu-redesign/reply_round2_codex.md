[Summary]
- v2는 코어 학습 루프 중심으로 잘 수렴했지만, MVP에서 남은 리스크는 “콘텐츠 운영 부담”과 “첫 30초 가치 전달”입니다.

[Findings]
- 데일리 챌린지는 로컬 시드 구현 가능 여부와 별개로 MVP 우선순위가 낮습니다. 기술비는 낮아도 콘텐츠 다양성, 난이도 편차, 반복 체감 관리가 필요해서 초기 리텐션보다 유지보수 리스크가 먼저 생깁니다.
- 기본 테마는 라이트가 맞습니다. “모던”은 다크만의 속성이 아니라 대비, 여백, 타이포, 색 체계로도 만들 수 있고, 아동 대상 서비스에서는 가독성과 진입 장벽이 라이트에서 더 안정적입니다.
- 통계를 독립 탭으로 두면 정보 구조가 과합니다. MVP에서는 홈과 프로필에 요약만 두고, 통계는 나중에 분리하는 편이 탐색 비용이 낮습니다.
- 하단 탭 4개 축소는 적절하지만, “맵”이 실제로 다음 학습 결정을 바꾸지 못하면 별도 탭일 이유가 약합니다. 지도 기능이 단순 시각화 수준이면 홈 안으로 흡수하는 편이 낫습니다.
- 첫 세션 30초 내 가치 전달은 MVP에 필요합니다. 이 서비스는 초반에 “바로 풀 수 있다”는 경험이 없으면 학습앱이 아니라 설정앱처럼 보이기 쉽습니다.
- 현재 v2는 코어 기능은 충분히 줄였지만, 실패 후 복귀 설계가 아직 약합니다. 오답 직후 무엇이 바뀌는지, 다음 문제에 어떻게 연결되는지가 명시돼야 학습 체감이 생깁니다.

[Disposition Hint]
- Q1: 데일리 챌린지는 **MVP 제외** 권장입니다. 포함하더라도 별도 진입점 없이 홈의 보조 배너 수준이 맞습니다.
- Q2: 기본은 **라이트 + 다크 토글**이 맞습니다. 다크 기본은 “모던” 인상은 강하지만 아동 접근성과 첫 사용 가독성을 해칩니다.
- Q3: 탭은 **4개 유지**를 권장합니다. 통계는 홈/프로필 요약으로 흡수하세요.

[Open Questions]
- 첫 진입 시 사용자가 즉시 풀 문제를 만나도록 할지, 짧은 설명 1개를 먼저 둘지 확정이 필요합니다.
- “맵”이 실제 의사결정 기능인지, 단순 진도 시각화인지 정의가 필요합니다.
- MVP의 최소 성공 기준을 “첫 세션 완료율”로 볼지 “둘째 날 재방문율”로 볼지 정해야 합니다.
