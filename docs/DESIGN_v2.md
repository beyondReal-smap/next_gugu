# 구구단 학습 앱 재설계 (DESIGN v2 — Round1 합의 반영)

> 목표 1문장: **"구구단을 게임처럼 반복 학습하게 만들어, 서버 없이도 매일 다시 들어오고 싶은 모던한 학습 앱"을 새로 만든다.**

변경 요약(v1→v2): 서버 의존 기능 전부 후순위 분리, 타깃·톤 좁힘, MVP 모드 축소, 스택 신규 도입 최소화, 풀 SRS→경량 오답가중, 브랜드/광고 정리 명시.

## 1. 타깃 & 톤 (쟁점 #2 반영)
- **1차 타깃 단일화**: **초등 4~6학년 + 중등 1학년** — 구구단을 "외우고, 빠르게 만드는" 핵심층.
- **2차(분리)**: 성인 두뇌 트레이닝 → 별도 진입("빠른 연산" 모드)로 두어 메인 톤 분열 방지.
- **톤 = "차갑지 않은 모던"**: 유아틱(파스텔·이모지 남발·토이폰트) 배제하되, 다크의 차가움도 완화.
  - 라이트/다크 **둘 다 지원**, **기본 라이트**(아동 접근성·밝은 동기) + 다크 토글.
  - 무채색 베이스 + **생기 있는 액센트 1색**(예: 일렉트릭 라임/바이올렛) + 성공/위험 보조색.
  - 캐릭터·이모지 대신 **추상 기하 마스코트(미니멀)** 또는 생략, 성장 시각화로 동기 부여.
  - 큰 산세리프 타이포 + 8pt 그리드 + 절제된 모션(150~250ms) + 햅틱.

## 2. 리텐션 전략 — 전부 로컬(서버 0) 구현 가능 (쟁점 #1 반영)
1. **스트릭**: 연속 플레이 일수. 메인 후크.
2. **데일리 골**: 하루 목표(정답 N개/세션 N회) 진행 링.
3. **데일리 챌린지(로컬 시드)**: 날짜를 시드로 고정 생성(서버 불필요). 매일 다른 미션 = 재방문 동기. *← v1 대비 MVP 유지 주장(쟁점 #3, 아래 근거)*
4. **레벨/XP**: 정답·속도·콤보로 XP, 레벨업 + 칭호.
5. **마스터리 맵**: 단별 숙련도(정확도·속도) 별점 1~3.
6. **업적(간소)**: 핵심 10여 종.
> 위클리 리그/랭킹·푸시·계정·동기화는 **서버 단계로 분리**(MVP 제외).

## 3. MVP 게임 모드 — 2(+1)종으로 축소 (쟁점 #3 반영)
- **학습(Practice)**: 단 선택, 즉시 피드백, 압박 없음. **오답 가중 + 최근오답 재출제**(경량, 쟁점 #5).
- **스피드런(Time Attack)**: 제한시간/문제수, 개인 베스트.
- **데일리 챌린지**: 날짜 시드 고정 미션(로컬). *서버 없이 구현 가능하므로 MVP 포함 주장.*
- (후순위) 서바이벌(하트), 위클리 리그.

## 4. 학습 출제 로직 (쟁점 #5 반영)
- 풀 SRS 폐기. 대신: **틀린 식 가중치 ↑ + 최근 오답 우선 재출제 + 단별 미숙련 식 우선**.
- 구구단 곱셈식은 약 64~100개로 적어, 이 경량 규칙으로 충분.

## 5. 정보 구조(IA) / 화면
- **홈**: 데일리 골 링 + 스트릭 + 레벨/XP 바 + "이어서/시작" CTA + 데일리 챌린지 카드.
- **세션**: 문제 + 키패드 + 진행바 + 콤보 + (스피드런)타이머.
- **결과**: 정답률·평균속도·획득 XP·레벨업·신규 별/배지 + "한 판 더".
- **맵**: 단별 마스터리 별점.
- **통계(간소)**: 정확도/평균속도 추이(스파크라인) + 약점 단 하이라이트.
- **프로필**: 레벨/칭호/배지.
- **하단 탭**: 홈 · 학습 · 맵 · 프로필 (4개; 통계는 프로필/홈에 통합해 단순화).

## 6. 비주얼 디자인
- 라이트 기본 + 다크 토글. 액센트 1색 + 보조색. 채도 절제, 이모지·토이폰트 배제.
- 데이터 시각화(링/스파크라인/단별 히트맵)로 "성장이 보이는" 모던함.
- 카드 + 8pt 그리드 + 넉넉한 여백 + 명확한 위계 + 마이크로 인터랙션.

## 7. 기술 스택 (쟁점 #4 반영 — 신규 도입 최소화)
- Next.js 15 App Router + TS + Tailwind.
- **상태**: React Context + useReducer + localStorage 영속화 훅(기존 패턴 재사용). **Zustand 미도입**.
- **애니메이션**: Framer Motion(기존).
- **차트**: 경량 **SVG 직접 구현**(Recharts 미도입).
- **PWA**: 기존 `next-pwa` 활용(설치/오프라인). **푸시는 서버 단계**(특히 iOS 웹푸시 제약).
- **배포**: **정적 export 유지**(pm2 `serve` → `next.smap.site:5104` + Cloudflare Tunnel). 서버 기능 도입 시 재검토.
- **구조**: feature 폴더(`features/session|progress|map|daily`) + `components/ui` + `lib`(도메인). 거대 컴포넌트 금지.

## 8. 브랜드 / 광고 (쟁점 #6 — 사용자 결정 필요)
- 새 제품 정체성("구구단 학습 게임"). 현행 SMAP 홍보 배너·구매유도·롤링배너는 **MVP에서 제거 또는 푸터 1개로 최소화** 제안.
- *결정 필요*: (a) SMAP 자회사형(브랜드 연계·배너 유지) vs (b) 독립 학습 제품(배너 제거). → 사용자 승인 항목.

## 9. 성공 지표 (Codex Open Q 반영)
- 주요: **D7 재방문율 + 스트릭 유지율**. 보조: 오답 재도전율(학습 효과).

## 10. 로드맵
- **MVP(로컬, 정적 export)**: 홈/학습(오답가중)/스피드런/데일리챌린지/스트릭/데일리골/레벨·XP/마스터리맵/업적/간소통계/라이트·다크.
- **v1.1(로컬)**: 서바이벌, 통계 강화.
- **v2(서버 도입, 순서: 계정 → 동기화 → 리그 → 푸시)**: 위클리 리그/랭킹, 크로스기기 동기화, 푸시 알림.

## 11. 남은 쟁점(→ Round2 재검토)

- (Q2) 기본 테마를 라이트로(아동 접근) 두는 절충이 "모던" 요구와 충돌하지 않나? 다크 기본이 더 맞나?
- (Q3) 하단 탭 4개(통계 통합) 단순화가 적절한가?
