# 구구단 학습 앱 재설계 — 최종 설계 (DESIGN final)

> 목표: **"구구단을 게임처럼 반복 학습하게 만들어, 서버 없이도 매일 다시 들어오고 싶은 모던한 학습 앱"을 새로 만든다.**
> 본 문서는 Claude 초안 + Codex 2라운드 토론으로 수렴한 최종 설계다. (근거: `.collab-loop/20260615-gugu-redesign/`)

## 1. 타깃 & 톤
- **1차 타깃(단일)**: 초등 4~6학년 + 중1 — 구구단을 외우고 속도를 올리는 핵심층.
- **2차(분리)**: 성인 두뇌 트레이닝 → "빠른 연산" 모드로 분리(메인 톤 분열 방지).
- **톤 = 차갑지 않은 모던**: 유아틱(파스텔·이모지 남발·토이폰트) 배제.
  - **라이트 기본 + 다크 토글**(모던함은 대비·여백·타이포·색 체계로 구현; 다크 강요 X — 아동 가독성/진입장벽 고려).
  - 무채색 베이스 + 생기 있는 액센트 1색 + 성공/위험 보조색(채도 절제).
  - 이모지·캐릭터 남발 대신 미니멀 기하 요소 + **성장 시각화**로 동기 부여.
  - 큰 산세리프 타이포 + 8pt 그리드 + 절제된 모션(150~250ms) + 햅틱.

## 2. 리텐션 (전부 로컬, 서버 0)
1. **스트릭** — 연속 플레이 일수(메인 후크)
2. **데일리 골** — 하루 목표 진행 링(정답 N개/세션 N회)
3. **레벨/XP** — 정답·속도·콤보로 XP, 레벨업 + 칭호
4. **마스터리(단별 별점)** — 단 숙련도(정확도·속도) 1~3별
5. **업적(간소 10여 종)**
> 데일리 챌린지·위클리 리그·랭킹·푸시·계정·동기화 = **서버 단계로 분리(MVP 제외)**.

## 3. MVP 게임 모드 — 2종
- **학습(Practice)**: 단/범위 선택, 즉시 피드백, 압박 없음. **오답 가중 + 최근오답 재출제 + 미숙련 식 우선**.
- **스피드런(Time Attack)**: 제한시간·문제수, 개인 베스트 갱신.
- (후순위) 서바이벌(하트), 데일리 챌린지, 위클리 리그, 성인 빠른연산.

## 4. 출제/학습 로직 (경량)
- 풀 SRS 폐기. 곱셈식 약 64~100개 도메인에 맞춰 **오답 가중치 + 최근 오답 우선 + 단별 미숙련 우선** 규칙.
- **실패 후 복귀 루프(필수)**: 오답 시 ① 즉시 정답 표시 + 해당 식 강조, ② 오답풀 등록(다음 출제 가중·즉시 인근 재등장), ③ 세션 종료 후 **"틀린 것만 다시 풀기"** 제공. → 오답 직후 무엇이 바뀌는지 명확히.

## 5. 온보딩 (MVP 필수)
- 최초 진입 시 **설명 최소 + 즉시 샘플 세션(2~3문제)**로 "바로 풀 수 있다"를 30초 내 체험 → 이후 홈. (설정앱처럼 보이지 않게)

## 6. 정보 구조(IA) — 탭 3개
- **홈**: 데일리 골 링 + 스트릭 + 레벨/XP 바 + "이어서/빠른 시작" CTA. (통계 요약 소형 위젯)
- **학습**: 모드 선택(학습/스피드런) + **단 맵(별점 + 단 선택=세션 진입)** 통합. (맵이 곧 학습 결정 기능)
- **프로필**: 레벨/칭호/배지 + 통계 요약(정확도·평균속도 스파크라인 + 약점 단 하이라이트).
- **세션/결과**: 풀스크린(탭 밖). 결과 = 정답률·평균속도·획득XP·레벨업·신규 별/배지 + "한 판 더"/"틀린 것만".

## 7. 비주얼
- 라이트 기본 + 다크 토글. 액센트 1색 + 보조색. 이모지/토이폰트 배제.
- 데이터 시각화(링/스파크라인/단별 히트맵)로 "성장이 보이는" 모던함.
- 카드 + 8pt 그리드 + 여백 + 위계 + 마이크로 인터랙션.

## 8. 기술 스택 (신규 도입 최소)
- Next.js 15 App Router + TS + Tailwind.
- 상태: **React Context + useReducer + localStorage 영속화 훅**(Zustand 미도입).
- 애니메이션: Framer Motion(기존). 차트: **SVG 직접 구현**(Recharts 미도입).
- PWA: 기존 `next-pwa` 활용(설치/오프라인). 푸시는 서버 단계.
- 배포: **정적 export 유지**(pm2 `serve` → `next.smap.site:5104` + Cloudflare Tunnel).
- 구조: feature 폴더(`features/session|progress|map|profile`) + `components/ui` + `lib`(도메인). 거대 컴포넌트 금지.

## 9. 성공 지표
- 주요: **D7 재방문율 + 스트릭 유지율**. 보조: 오답 재도전율, 첫 세션 완료율.

## 10. 로드맵
- **MVP(로컬/정적 export)**: 온보딩 · 홈 · 학습(오답가중) · 스피드런 · 스트릭 · 데일리골 · 레벨/XP · 마스터리맵 · 업적 · 통계요약 · 라이트/다크 · 실패복귀 루프.
- **v1.1(로컬)**: 서바이벌, 데일리 챌린지(로컬 시드), 통계 강화.
- **v2(서버, 순서: 계정 → 동기화 → 리그 → 푸시)**: 위클리 리그/랭킹, 크로스기기 동기화, 푸시 알림.

## 11. 사용자 결정 필요 (구현 전 확정)
- **브랜드/광고**: (a) SMAP 자회사형(브랜드 연계·홍보 배너 유지) vs (b) 독립 학습 제품(배너 제거). 기본 제안은 (b) 또는 푸터 1개로 최소화.
- **구현 방식**: 기존 `next_gugu` 클린 재구성 vs 신규 디렉토리. 배포 경로(5104)는 유지.
