# 구구단 학습 앱 전면 재설계 (DESIGN v1)

> 목표 1문장: **"구구단을 게임처럼 반복 학습하게 만들어, 매일 다시 들어오고 싶은 모던한 학습 앱"을 새로 만든다.**

## 0. 재설계 배경
기존 `next_gugu`는 단일 거대 컴포넌트(page.tsx 1700+줄)에 연습/타임어택 두 모드만 있고, 디자인이 아동용(파스텔·이모지·둥근 토이룩)으로 치우쳐 있다. 사용자는 (1) 레벨/게임 요소로 **리텐션(재방문)** 강화, (2) **모던하고 세련된** 비주얼을 원한다. 따라서 코드/IA/디자인을 새로 설계한다.

## 1. 타깃 & 포지셔닝
- **1차 타깃**: 구구단을 익히거나 속도를 끌어올리려는 **초등 3~6학년 + 중등 저학년**.
- **2차 타깃**: 암산/두뇌 트레이닝을 원하는 **성인**(라이트 게임).
- **포지셔닝**: "구구단판 듀오링고" — 짧은 세션(1~3분), 매일 조금씩, 성장이 눈에 보이는 학습 게임.
- **디자인 톤**: 유아틱 배제. 차분한 다크 베이스 + 절제된 액센트 1~2색 + 큰 타이포 + 데이터 시각화. 친근하되 세련됨(스포티/테크).

## 2. 리텐션 전략 (왜 매일 들어오나)
1. **데일리 골 + 스트릭**: 하루 목표(예: 정답 20개 또는 3세션) 달성 시 스트릭 +1. 스트릭 유지가 핵심 후크.
2. **레벨 & XP**: 정답/속도/콤보로 XP 획득, 레벨업. 레벨에 칭호.
3. **마스터리 맵**: 2~9단(확장 시 ~19단)을 단계적으로 "마스터" — 단별 숙련도(정확도·속도) 별점.
4. **위클리 리그/랭킹**: 주간 XP로 리그 승강(브론즈→실버→골드…). *서버 필요*.
5. **데일리 챌린지**: 매일 바뀌는 미션(예: "7단 10문제 8초 안에").
6. **푸시 알림**: 스트릭 유지 리마인더, 데일리 챌린지 알림. *서버/PWA 필요*.
7. **성장 대시보드**: 정확도·평균 응답속도·약점 단(段) 추이 그래프.

## 3. 게임 메커니즘
- **세션 구조**: 한 세션 = 10~15문제. 짧고 빠르게.
- **모드**:
  - *학습(Practice)*: 단 선택, 틀린 문제 즉시 피드백, 압박 없음.
  - *스피드런(Time Attack)*: 제한시간/문제수, 개인 베스트 갱신.
  - *서바이벌*: 하트 3개, 틀리면 하트 감소(긴장감 + 세션 길이 자연 제한).
  - *데일리 챌린지*: 고정 시드 미션.
- **콤보/배수**: 연속 정답 시 XP 배수. 빠른 응답 보너스.
- **약점 집중(Adaptive)**: 자주 틀리는 곱셈식을 가중 출제(스페이스드 리피티션 라이트).
- **보상**: XP, 별, 배지(업적), 리그 승급. 가챠/과금 없음(학습 앱 신뢰).

## 4. 정보 구조(IA) / 화면
- **홈(대시보드)**: 오늘의 골 진행 링, 스트릭, 레벨/XP 바, "이어서 학습" CTA, 데일리 챌린지 카드.
- **학습 진입**: 모드 선택 → 단/난이도 선택 → 세션.
- **세션 화면**: 문제 + 입력(키패드/선택지), 진행바, 콤보, 남은 하트/시간.
- **결과 화면**: 정답률·속도·획득 XP·레벨업·새 배지, "한 판 더" / "공유".
- **맵**: 단별 마스터리 + 별점.
- **통계**: 성장 그래프, 약점 단.
- **프로필**: 레벨/칭호/배지 컬렉션/리그.
- **하단 탭**: 홈 · 학습 · 맵 · 통계 · 프로필 (5개, 모바일 우선).

## 5. 비주얼 디자인 방향
- **모드**: 다크 우선(딥 네이비/차콜) + 라이트 토글.
- **컬러**: 무채색 베이스 + 액센트 1색(예: 일렉트릭 라임 또는 시안) + 보조 1색(위험/성공). 채도 절제.
- **타이포**: 숫자 강조용 가변 산세리프(예: Inter/Geist) + 큰 디스플레이 수치. 라운드 토이폰트 배제.
- **모션**: 짧고 정교한 마이크로 인터랙션(150~250ms), 과한 무지개/폭죽 대신 절제된 피드백(스케일/글로우/햅틱).
- **데이터 시각화**: 링 프로그레스, 스파크라인, 히트맵(약점 단). "성장이 보이는" 화면이 곧 모던함.
- **레이아웃**: 카드 + 8pt 그리드, 넉넉한 여백, 명확한 위계.

## 6. 기술 스택 / 아키텍처
- **프레임워크**: Next.js 15 (App Router) + TypeScript + Tailwind.
- **상태관리**: Zustand(경량) — 게임/진행 상태. 영속화는 `persist` 미들웨어.
- **애니메이션**: Framer Motion.
- **차트**: 경량 SVG 직접 구현 또는 Recharts(번들 고려).
- **영속화**: MVP는 localStorage. 리그/랭킹/푸시/크로스기기 동기화는 **백엔드 필요** → Supabase(Postgres + Auth) 단계적 도입.
- **배포**: 현 인프라 호환을 위해 **정적 export 유지**(pm2 `serve` → `next.smap.site`). 단, 서버 기능(랭킹/푸시)은 정적 export와 충돌 → API는 별도(Supabase Edge / 기존 smap api) 또는 export 해제 후 `next start` 전환 필요.
- **PWA**: 설치 + 오프라인 + 푸시(리텐션 핵심). 단 iOS 웹푸시 제약 존재.
- **구조**: feature 폴더(`features/session`, `features/progress`, `features/map`…) + 공용 `components/ui` + `lib`(도메인 로직). 거대 컴포넌트 금지.

## 7. MVP 범위 vs 확장
- **MVP(로컬 only, 정적 export 유지)**: 홈 대시보드, 학습/스피드런/서바이벌, 마스터리 맵, XP/레벨, 스트릭, 데일리 골, 업적, 통계, 다크/라이트. → 서버 없이 리텐션 핵심 일부 확보.
- **확장(서버 도입)**: 계정, 위클리 리그/랭킹, 푸시 알림, 크로스기기 동기화, 데일리 챌린지 서버 시드.

## 8. 마이그레이션
- 기존 `next_gugu`를 새 구조로 재작성(별도 디렉토리 또는 클린 재구성). 배포 경로(`next.smap.site`, 포트 5104)는 유지.
- 기존 localStorage 키(`multiplicationGame`, `guguGameProgress`)와 충돌 없이 새 스키마 사용(필요 시 마이그레이션 함수).

## 9. 핵심 리스크 / 열린 질문
- (R1) 리텐션 기능 다수가 **서버 의존**인데 MVP는 정적 export → 어디까지 로컬로 가능한가? export 해제 트레이드오프는?
- (R2) 타깃이 아동+성인으로 넓음 → 디자인/난이도 톤이 분열되지 않나?
- (R3) "모던/다크"가 아동 학습 동기엔 오히려 차갑지 않나? (듀오링고는 밝고 캐릭터 중심)
- (R4) 스택 신규 도입(Zustand/Recharts/PWA)의 번들·복잡도 대비 효용?
- (R5) 푸시 알림 없이 리텐션 후크가 충분한가?
